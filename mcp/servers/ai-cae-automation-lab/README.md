# AI CAE Automation Lab MCP Server Package

This folder is the product-facing MCP bridge package for this repository. It is
kept under `mcp/` so GitHub readers can see a complete server layout, while the
implementation reuses the tested core package under `src/ai_cae_lab`.

## What This Server Bridges

- Abaqus: noGUI script execution and input-deck submission
- Ansys Fluent: journal execution and evidence capture
- Ansys Workbench: journal execution and installation checks
- COMSOL Multiphysics: Java API cube smoke, compile, batch, Java-to-MPH flow
- MATLAB: batch script execution
- OpenFOAM: selected command execution for local/WSL workflows
- ParaView: pvpython postprocessing execution
- PCSCHEMATIC Automation: installation and COM/OLE evidence checks

## First Run

From the repository root:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[mcp]"
.\.venv\Scripts\ai-cae-toolbox.exe setup
. .\private\activate-ai-cae.ps1
.\.venv\Scripts\python.exe mcp\servers\ai-cae-automation-lab\server.py
```

For Codex, copy the TOML generated by `ai-cae-toolbox setup` from
`private/codex-mcp.local.toml` into the local Codex config.

## Layout

| Path | Purpose |
|---|---|
| `server.py` | Runnable MCP server wrapper that imports `ai_cae_lab.mcp_server` |
| `tools/` | Tool-group wrappers for discovery, runs, evidence, and solver bridges |
| `schemas/` | JSON schemas for local config, run requests, and evidence summaries |
| `profiles/` | Solver capability profiles for agent planning and UI generation |
| `examples/` | Codex config, local config, and smoke workflow examples |
