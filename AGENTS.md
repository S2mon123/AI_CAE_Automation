# AI CAE Automation Lab Agent Instructions

For normal solver tasks, do not scan the whole repository.

1. Call `solver_context_scope` or run `ai-cae-toolbox context-scope --solver <solver>` first.
2. Read only the returned solver-specific files/directories unless the user asks for a repository-wide audit.
3. Resolve executables from explicit user input, environment variables, or `private/ai-cae.local.json` generated by `ai-cae-toolbox setup`. If paths are missing, call `discover_toolchains`/`setup_local_toolchain` instead of scanning the whole repository.
4. Treat paths in `configs/*.template.json`, `configs/*.example.json`, README files, and MCP examples as placeholders.
5. Write generated models, logs, exports, and solver outputs under a run directory, then call `scan_run_evidence` on that run directory only.
6. Never claim solver success from file presence alone; use logs and evidence grading.

Public source files must not contain author-machine paths, license files, commercial manuals, private models, or large generated solver artifacts.
